
# Руководство разработчика для разворачивания текущего веб-приложения для предсказания группы ткани мышей

# P.S. полноценное описание обработки данных и решения описывается в отчете в формате pdf

Данное руководство предназначено для пользователей ПК с операционной системой Windows.

## 1. Запуск веб-сервиса

Для запуска веб-сервиса необходимо выполнить следующие шаги.

### Шаг 1

Установите на свой ПК Docker Desktop. Для этого перейдите по ссылке:
https://www.docker.com/products/docker-desktop/

Далее в соответствии с вашей операционной системой выберите установщик и завершите установку.

### Шаг 2

Запустите Docker Desktop и дождитесь полной инициализации программы. При необходимости выполните вход в аккаунт или завершите первоначальную настройку.

### Шаг 4

Проект состоит из двух частей:

- front-end, который собирается из отдельного Dockerfile и работает на порту 5173;
- back-end, который собирается из отдельного Dockerfile и работает на порту 3001.

В проекте также используется файл `docker-compose.yml`, который запускает оба сервиса вместе.

Согласно предоставленным файлам:

- front-end сервис собирается из контекста `./frontend` и публикуется на порту `5173:5173`;
- back-end сервис собирается из контекста `./backend`, запускает команду `node server.js` и публикуется на порту `3001:3001`;
- front-end зависит от back-end через `depends_on`;
- оба сервиса подключены к сети `app-network`. 

### Шаг 6

Для стандартного запуска проекта рекомендуется использовать Docker Compose. Находясь в корневой папке проекта, выполните команду:

```bash
docker compose -f docker-compose-3.yml up --build
```

Эта команда соберёт front-end и back-end контейнеры и затем запустит их вместе.

Если нужно запустить проект в фоновом режиме, используйте:

```bash
docker compose -f docker-compose-3.yml up --build -d
```

### Шаг 7

После запуска откройте Docker Desktop и перейдите в раздел Containers. В списке должны появиться два работающих контейнера: `frontend` и `backend`.

### Шаг 8

После успешного запуска приложение будет доступно по следующим адресам:

- front-end: `http://127.0.0.1:5173`
- back-end: `http://127.0.0.1:3001`

Во front-end контейнере автоматически создаётся `.env` файл со значением `VITE_BACKEND_URL=http://backend:3001`, поэтому пользовательский интерфейс обращается к back-end по имени сервиса внутри docker-сети.

### Шаг 9

Для остановки проекта выполните команду:

```bash
docker compose -f docker-compose-3.yml down
```

Если проект был запущен не в фоне, его также можно остановить сочетанием клавиш `CTRL + C`, после чего рекомендуется выполнить команду остановки compose.

## 3. Ручная сборка без compose

Если требуется отдельно собрать и запустить front-end и back-end без Docker Compose, можно выполнить команды вручную.

### Сборка front-end

На основе предоставленного Dockerfile front-end использует образ `node:24-alpine`, рабочую директорию `/app`, устанавливает зависимости через `npm install`, создаёт `.env` с адресом back-end и запускает приложение на порту 5173.

Команда сборки:

```bash
docker build -t frontend-local:latest -f ./frontend/Dockerfile ./frontend
```

### Сборка back-end

На основе предоставленного Dockerfile back-end использует образ `python:3.11-slim`, дополнительно устанавливает Node.js 24, Python-зависимости, npm-пакеты и открывает порт 3001.

Команда сборки:

```bash
docker build -t backend-local:latest -f ./backend/Dockerfile ./backend
```

### Запуск back-end

```bash
docker run --rm -it -p 3001:3001 backend-local:latest
```

### Запуск front-end

```bash
docker run --rm -it -p 5173:5173 frontend-local:latest
```

Для стандартной эксплуатации проекта рекомендуется использовать именно Docker Compose, так как он сразу поднимает оба сервиса, сеть, тома и зависимости между контейнерами.

## 4. Дополнительные сведения

В конфигурации Docker Compose для back-end подключены тома `backend_uploads`, `./backend/processed_results:/app/processed_results` и `./backend/models:/app/models`, поэтому результаты обработки и модели хранятся отдельно от контейнера.

Для front-end и back-end также подключаются рабочие директории проекта через volumes, что удобно для локальной разработки и мгновенного применения изменений в коде.

В compose-файле присутствует секция `develop.watch`, которая используется для синхронизации файлов и пересборки при изменениях в режиме разработки.

## 5. Типовые ошибки

### Ошибка: git is not recognized

Причина: Git не установлен или не добавлен в PATH.

Решение: установить Git и перезапустить терминал.

### Ошибка: repository not found

Причина: неверный URL репозитория или нет прав доступа к нему.

Решение: проверить адрес репозитория и доступы.

### Ошибка: failed to solve during docker build

Причина: ошибка в одном из Dockerfile, не хватает файлов проекта или неверно указан build context.

Решение: проверить структуру каталогов `frontend` и `backend`, а также соответствующие Dockerfile.

### Ошибка: service frontend or backend failed to start

Причина: ошибка в приложении, зависимостях или конфликт портов.

Решение: посмотреть логи через команду:

```bash
docker compose -f docker-compose-3.yml logs
```

### Ошибка: port already allocated

Причина: порты 5173 или 3001 уже заняты другим приложением.

Решение: изменить проброс портов в compose-файле.
